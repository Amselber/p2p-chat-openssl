# p2p-chat-openssl

Децентрализованный чат для локальной сети. В разработке.

## Цель

Общение без сервера. Все узлы равноправны.
Обнаружение через UDP, защита через TLS, идентификация через fingerprint сертификата.

## Быстрый старт 
```bash
    git clone --recursive https://github.com/Amselber/p2p-chat-openssl.git
    cd p2p-chat-openssl/
```
Все команды выполняются из корневой директории проекта.
1. Настройте сеть. По умолчанию для работы udp-multicast используется адрес 239.255.0.1/24
 - должен существовать маршрут к данной подсети: 
```
sudo ip route add 239.255.0.0/24 dev <interface>
```

2.  Для запуска необходимо сгенерировать сертификаты и указать путь к ним в p2pchat.ini
 - Запуск с настройками по умолчанию:
``` bash
    ./certs/gen_certs.sh certs/ client
    make run
```
 - Внимание! Если вы хотите запустить несколько клиентов - сгенерируйте ключи с разными именами и укажите пути к ним в p2pchat.ini для каждого клиента (вместо client выберите имя) корневой сертификат rootCA.crt должен быть общим для всех. 

 - Пример генерации сертификатов для нескольких пользователей:
 ```
    ./certs/gen_certs.sh certs/ alice bob jack
 ```
 - Посмотреть пример параллельной работы нескольких клиентов можно по инструкции из следующего подраздела.
 После выполнения команды
 ```bash
    make itest
 ```
 в каталоге ```bin/test``` появятся директории ```chat1/ chat2/ chat3/ chat4/``` соответствующие разным экземплярам для 4х клиентов.

## P2P Chat — конфигурация

Конфигурационный файл `p2pchat.conf` в формате INI.

### Секция `[network]`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `multicast_addr` | string | `239.255.0.1` | Адрес multicast-группы для обнаружения узлов |
| `multicast_port` | uint16 | `9000` | Порт multicast-группы |
| `hello_interval` | int | `5` | Интервал отправки HELLO-пакетов (секунды) |

### Секция `[security]`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `ca_cert` | path | `certs/rootCA.crt` | Путь к корневому сертификату |
| `my_cert` | path | `certs/client.crt` | Путь к своему сертификату |
| `my_key` | path | `certs/client.key` | Путь к своему приватному ключу |

### Секция `[logging]`

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `level` | string | `info` | Уровень: `debug`, `info`, `warn`, `error`, `fatal` |
| `console` | bool | `yes` | Вывод логов в консоль |
| `file` | path | `p2pchat.log` | Путь к файлу лога |

### Пример

```ini
[network]
multicast_addr = 239.255.0.1
multicast_port = 9000
hello_interval = 5

[security]
ca_cert  = certs/rootCA.crt
my_cert  = certs/alice.crt
my_key   = certs/alice.key

[logging]
level   = info
console = yes
file    = p2pchat.log
```

### Примечания

- Все пути относительны от директории запуска
- При отсутствии файла используются значения по умолчанию
- \# и ; в начале строки — комментарии
- Идентификатор узла (fingerprint) и имя (CN) извлекаются из сертификата автоматически

### Команды чата
Для подсказки по командам после запуска введите
```
    /help
```

### Параметры командной строки
Выбор конфигурационного файла
```bash
./p2pchat -c ./p2pchat.ini
```
Подсказки и версия
```
./p2pchat help
./p2pchat version
```
